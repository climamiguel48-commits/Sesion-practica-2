Control de calidad de datos meteorológicos

Prof. Miguel Silva

¿Por qué control de calidad?

  • Los datos no son perfectos

  • Prescencia de errores de medición, transmisión o procesamiento

  • Datos faltantes o valores extremos no realistas

Consecuencias de no controlar calidad

  • Estadísticas sesgadas (medias, máximos, tendencias)

  • Gráficos engañosos

  • Decisiones agrícolas incorrectas

Objetivo de la clase

Aprender a:

  • Identificar valores faltantes en series climáticas

  • Verificar rangos plausibles por variable

  • Detectar incoherencias entre variables

  • Identificar outliers usando método IQR

  • Generar un dataset limpio para análisis posteriores

Verificamos los rangos plausibles

Cada variable tiene límites físicos:

Variable Rango Unidad
🌡️ T_media 15 a 35 °C
🔥 T_max 20 a 45 °C
❄️ T_min 10 a 28 °C
💧 Precipitación 0 a 300 mm/día
💨 HR 50 a 100 %
🌬️ Viento 0 a 30 m/s
☀️ Radiación 100 a 350 W/m²

Los rangos pueden variar de acuerdo a:

  • La escala de medición; para escalas mensuales, el valor máximo de precipitación puede variar.

  • Los rangos pueden cambiar dependiendo de la climatología de un sitio particular. No se pueden aplicar estos mismos criterios en zonas templadas.

Detectamos outlier con IQR

¿Qué es un outlier?

Un outlier (valor atípico) es un dato que se aleja significativamente del resto de las observaciones.

Ejemplo: Si la temperatura media en un mes tropical es 25°C y de repente aparece un registro de 45°C, ese es un outlier.

¿Qué es el IQR?

IQR = Rango Intercuartil (Interquartile Range)

Es la distancia entre el percentil 75 (Q3) y el percentil 25 (Q1):

IQR = Q3 - Q1

¿Cómo detectamos outliers?

Usamos la regla del IQR:

Límite inferior = Q1 - k × IQR Límite superior = Q3 + k × IQR

Donde k es un factor (nosotros usamos 3)

Un outlier es cualquier valor fuera de estos límites.

¿Qué pasa con la precipitación?

La precipitación tiene un problema: no sigue una distribución normal (muchos días con cero y pocos días con mucha lluvia), por lo tanto, no aplicaremos detección de outliers con esta variable.

Procedemos a detectar los outliers

# Calcular IQR para T_media
Q1 <- quantile(datos$T_media, 0.25, na.rm = TRUE)
Q3 <- quantile(datos$T_media, 0.75, na.rm = TRUE)
IQR <- Q3 - Q1
limite_inferior <- Q1 - 3 * IQR
limite_superior <- Q3 + 3 * IQR
datos$T_media_outlier <- datos$T_media < limite_inferior | datos$T_media > limite_superior
cat("T_media - Outliers:", sum(datos$T_media_outlier, na.rm = TRUE), "\n")

# Calcular IQR para HR
Q1 <- quantile(datos$HR, 0.25, na.rm = TRUE)
Q3 <- quantile(datos$HR, 0.75, na.rm = TRUE)
IQR <- Q3 - Q1
limite_inferior <- Q1 - 3 * IQR
limite_superior <- Q3 + 3 * IQR
datos$HR_outlier <- datos$HR < limite_inferior | datos$HR > limite_superior
cat("HR - Outliers:", sum(datos$HR_outlier, na.rm = TRUE), "\n")...
